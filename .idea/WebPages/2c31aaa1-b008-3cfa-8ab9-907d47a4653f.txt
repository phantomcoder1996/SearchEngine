           <div class="post-text" itemprop="text"> 
            <p>You need to upgrade your local postgres to get the last security patch from the 2018-03-01, like Heroku did the 1st march. You need one of the last releases 10.3, 9.6.8, 9.5.12, 9.4.17, and 9.3.22. The security patch can be found here <a href="https://www.postgresql.org/about/news/1834/" rel="noreferrer">https://www.postgresql.org/about/news/1834/</a>. It seems the patch modified pg_dump, that's probably why we can't use pg_restore anymore without that patch for the dump of Heroku (with the patch applied).</p>

            <p>I ran into this same issue today, I don't know if something changed on Heroku's side, because locally I'm running 9.6 but my database on Heroku is 9.4, so it didn't seem to be about the version point difference (it was working until today.)</p> 
            <p>If you're using the Postgres.app (<a href="https://postgresapp.com/" rel="noreferrer">https://postgresapp.com/</a>) on a Mac make sure you're running <code>v2.1.3</code> (as of this writing that is the newest version). When you upgrade the Postgres.app you get the newest patch release of Postgres, (which is likely causing the mismatch you're currently experiencing - Heroku has upgraded for the latest security patch, and your local machine may not have it). </p> 
            <blockquote> 
             <p>I had to upgrade Postgres.app from 2.1.0 to 2.1.3 and it solved the problem.</p> 
            </blockquote>

sudo apt-get update
</code></pre>

            <p>I don't recommend upgrading your DB just to fix this as you want that to match whatever version you have in production.</p> 
            <p>This fixed it on Ubuntu 16.04, restoring from a Heroku database. </p> 
            <p>First, ensure you have the postgresql repo as diego mentioned. Then install the upgraded client.</p> 
            <p><code>sudo apt-get update &amp;&amp; sudo apt-get install postgresql-client-10</code></p> 
            <p>If you are Ubuntu 17.04 or 17.10, note that the PostgreSQL's zesty repo does <strong>not</strong> have the updated client's you need. You will want to use the xenial Postgres repo instead. </p>
