          <div class="post-text" itemprop="text"> 
           <p>I'm currently using <a href="https://developer.android.com/training/testing/espresso/index.html" rel="nofollow noreferrer">Espresso</a> to programmatically take screenshots of an Android app. I'm successfully using</p> 
           <pre><code>android.support.test.runner.screenshot.Screenshot
</code></pre> 
           <p>Problem is, given the following assumptions <a href="https://developer.android.com/training/testing/espresso/index.html" rel="nofollow noreferrer">from official Espresso docs</a>:</p> 
           <blockquote> 
            <ul> 
             <li>The message queue is empty. </li> 
             <li>There are no instances of AsyncTask currently executing a task. </li> 
             <li>All developer-defined idling resources are idle.</li> 
            </ul> 
            <p>By performing these checks, Espresso substantially increases the likelihood that only one UI action or assertion can occur at any given time. This capability gives you more reliable and dependable test results.</p> 
           </blockquote> 
           <p>I can't perform any test while any <code>AsyncTask</code> is running. I need to take some screenshots while my <code>AsyncTask</code> is running, but can't get my head around the docs on how to perform such task. <a href="https://stackoverflow.com/a/38385661/2378095">The most similar SO thread I found</a> doesn't seem to work, don't know if this is because the <code>AsyncTask</code> is too fast to take the screenshot.</p> 
           <pre><code>// Start the async task    
onView(withId(R.id.start_task_button)).perform(click());

// Then "press" the back button (in the ui thread of the app under test)
mActivityTestRule.runOnUiThread(new Runnable() {
    @Override
    public void run() {
        mScreenshotWatcher.captureScreenshot("screenshot1");
    }
});
</code></pre> 
           <p>Any clue about this? Is this the correct approach or should I implement <code>IdlingResource</code> in my <code>AsyncTask</code>?</p> 
           <p>thanks nicola</p>
