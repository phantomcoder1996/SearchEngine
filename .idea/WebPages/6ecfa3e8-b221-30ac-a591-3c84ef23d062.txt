        <h1>MongoDB – Where is the log file?</h1> 
        <div class="post-meta"> 
         <div class="post-meta">
          By 
          <a href="https://www.mkyong.com/author/mkyong/" title="mkyong class=" author" rel="author">mkyong</a> | 
          <time datetime="2017-08-27T07:51:14+00:00">August 27, 2017</time> | Viewed : 10,513 | 
          <span>+722 pv/w</span>
         </div> 
        </div> 
        <div id="post-content"> 
         <p>Default, MongoDB creates the log file at this path <code>/var/log/mongodb/mongodb.log</code>, if the log file is not found, please check with the MongoDB config file.</p> 
         <h2>logpath</h2> 
         <p>Check the MongoDB config file at <code>/etc/mongod.conf</code> or <code>/yourMongoDBpath/mongod.conf</code>, the <code>logpath</code> defined where to log.</p> 
         <div class="filename">
          /etc/mongod.conf
         </div> 
         <pre><code class="language-bash">
$ cat /etc/mongod.conf
# mongod.conf

# Where to store the data.

# Note: if you run mongodb as a non-root user (recommended) you may
# need to create and set permissions for this directory manually,
# e.g., if the parent directory isn't mutable by the mongodb user.
dbpath=/var/lib/mongodb3

#where to log
logpath=/var/log/mongodb/mongod.log

#...
</code></pre>
