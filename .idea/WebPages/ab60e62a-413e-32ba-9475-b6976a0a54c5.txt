           <div class="post-text" itemprop="text"> 
            <p>The same error is happening to me. Apparently, a new version of the <code>com.android.support:support-v4</code> library was released, and the plugin I'm using defines <code>com.android.support:support-v4:+</code> as dependency in <code>plugin.xml</code>. The <code>+</code> sign means that it will get the latest version (28.0.0), which seems seems to be incompatible with other plugins.</p> 
            <p>I was able to build a development version by changing all the plugin dependencies from <code>com.android.support:support-v4:+</code> to <code>com.android.support:support-v4:27.1.0</code>. Also, I executed <code>ionic cordova platform remove android</code> and <code>ionic cordova platform add android</code>. Hope it helps, at least for development.</p>

           <div class="post-text" itemprop="text"> 
            <p>Google released the new version <strong>28.0.0-alpha1</strong> of <strong>com.android.support:support-v4</strong> which is adding 2 new attributes(android:fontVariationSettings and android:ttcIndex). Some of the plugins are using the latest android-support libraries which results in unwanted incompatibilities.</p> 
            <p><strong>Option 1:</strong> Install <strong>cordova-android-support-gradle-release</strong> plugin.</p> 
            <p>Well documented plugin which "aligns various versions of the Android Support libraries specified by other plugins to a specific version". Tested without any destructive behavior. </p> 
            <pre><code>cordova plugin add cordova-android-support-gradle-release --fetch
</code></pre> 
            <p>Read the documentation for a full set of options: <a href="https://github.com/dpa99c/cordova-android-support-gradle-release" rel="noreferrer">Readme</a></p> 
            <p><strong>Option 2</strong>: Add next code snippet in <strong>build.gradle</strong> under <strong>platforms/android</strong></p> 
            <pre><code>/** 
IMPORTANT - Manually added
Problem: 8 March 2018 - Google released version support-v4:28.0.0-alpha1 
which breaks the project with following error: unable to find attribute 
android:fontVariationSettings and android:ttcIndex

Effect: Force a specific version of the library
*/

configurations.all {
    resolutionStrategy.force 'com.android.support:support-v4:27.1.0'
}
</code></pre> 
            <p><strong>Warning:</strong> code in <strong>build.gradle</strong> will be overwritten if you remove/add the Android platform. If you don't want to use the plugin for some reason or somehow is not working for you, instead create a hook and overwrite the file every time. Check 2nd comment <a href="https://stackoverflow.com/a/49174659/4423449">here</a>.</p> 
            <p><strong>If the problem is persistent you may try:</strong></p> 
            <pre><code>cordova platform rm android
cordova platform add android
</code></pre> 
            <p><strong>OR</strong></p> 
            <p>Make sure you don't have a <strong>previous version of the app installed</strong> on the device you test because you'll receive an ambiguous error when it tries to downgrade the existing version: "INSTALL_FAILED_VERSION_DOWNGRADE" and "UnhandledPromiseRejectionWarning: Unhandled promise rejection" </p>

            <p>It's strange, but it works when I add the below lines with the same versions.</p> 
            <p>This is my related lines in the <code>platforms/android/build.gradle</code> file:</p> 
            <pre><code>dependencies {
  compile fileTree(dir: 'libs', include: '*.jar')
  // SUB-PROJECT DEPENDENCIES START
  debugCompile(project(path: "CordovaLib", configuration: "debug"))
  releaseCompile(project(path: "CordovaLib", configuration: "release"))
  compile "com.android.support:support-v4:26.+"
  compile "com.android.support:appcompat-v7:26.+"
  // SUB-PROJECT DEPENDENCIES END
}

// ADDED THESE LINES
configurations.all {
  resolutionStrategy.force 'com.android.support:support-v4:26+'
}
</code></pre> 
            <p>In my project, the problem was occurred because of the 'cordova-plugin-crosswalk-webview' plugin.</p>

            <pre><code>gradlew -q dependencies app:dependencies --configuration debugAndroidTestCompileClasspath
</code></pre> 
            <p>Then fixed it with:</p> 
            <pre><code>configurations.all {
    resolutionStrategy {
        force 'com.android.support:support-compat:{the_same_version}'
        force 'com.android.support:appcompat-v7:{the_same_version}'
        force 'com.android.support:support-core-utils:{the_same_version}'
        force 'com.android.support:support-core-ui:{the_same_version}'
        force 'com.android.support:support-fragment:{the_same_version}'
        force 'com.android.support:support-annotations:{the_same_version}'
        ...
    }
}
</code></pre> 
            <p>Above code force the dependencies version.</p>

           <div class="post-text" itemprop="text"> 
            <p>Here's an easy way to fix it that will persist when the platform directory is rebuilt and there's no need to go through all your plugins to try and find a culprit. Create a file <code>build-extras.gradle</code> with these contents:</p> 
            <pre><code>configurations.all {
    resolutionStrategy {
        force 'com.android.support:support-v4:27.1.0'
    }
}
</code></pre> 
            <p>Then create the file <code>after_platform_add/010_copy_build_extras.js</code> with the following contents:</p> 
            <pre><code>#!/usr/bin/env node

var fs = require('fs');

var rootdir = process.argv[2];
var android_dir = `${rootdir}/platforms/android`;
var gradle_filename = 'build-extras.gradle';
var gradle_file = `${rootdir}/${gradle_filename}`;
if (fs.existsSync(android_dir) &amp;&amp; fs.existsSync(gradle_file)) {
  fs.createReadStream(gradle_file)
    .pipe(fs.createWriteStream(`${android_dir}/${gradle_filename}`));
}
</code></pre> 
            <p>Now recreate the android platform and it will use the pinned support library.</p>

            <p>I was having the same problem out of the blue yesterday. It started randomly, but from reading around, it looks like it is to do with an update as mentioned above by @cpro90. However, I tried and could not find where to make the necessary manual change.</p> 
            <p>Eventually I identified that the problem was being caused by my cordova-plugin-crosswalk-webview plugin. On GitHub, I found <a href="https://github.com/crosswalk-project/cordova-plugin-crosswalk-webview/issues/204" rel="nofollow noreferrer">the issue</a> on the plugin repro this morning, and it had over 520 views by lunch.</p> 
            <p>@UNUMObile suggested the following in the <code>build.gradle</code> file to force an earlier version globally:</p> 
            <pre><code>configurations.all {
    resolutionStrategy.force 'com.android.support:support-v4:24.0.0'
}
</code></pre> 
            <p>This worked immediately for me and may help others with other plugins that also have had their dependency on 'com.android.support:support-4:&lt;28. The new version 28 seems to be the issue.</p> 
            <p>I hope this helps someone move forward.</p>

            <p>Another aproach with same solution is create a hook. It's persistent (after platform reinstall), you can commit it and it don't require re-adding platform.</p> 
            <blockquote> 
             <p>%project%\scripts\android\android-support-version.js</p> 
            </blockquote> 
            <pre><code>#!/usr/bin/env node

var fs = require('fs');

function replace_strings_in_file(filename, replacementsObject) {
  if (fs.existsSync(filename)) {
    var data = fs.readFileSync(filename, 'utf8');
    Object.keys(replacementsObject).forEach(function (to_replace) {
      var replace_with = replacementsObject[to_replace];
      data = data.replace(to_replace, replace_with);
    });
    console.log(data);
    fs.writeFileSync(filename, data, 'utf8');
  } else {
    console.log('file not found');
  }
}

module.exports = function (context) {
  var rootdir = process.argv[2];
  if (rootdir) {
    replace_strings_in_file("platforms/android/project.properties", {'com.android.support:support-v4:+': 'com.android.support:support-v4:27.1.0'});
    console.log('com.android.support version fix');
  }
};
</code></pre> 
            <p>Init hook in config</p> 
            <blockquote> 
             <p>%project%\config.xml</p> 
            </blockquote> 
            <pre><code>...
&lt;platform name="android"&gt;
  &lt;hook src="scripts/android/android-support-version.js" type="before_build" /&gt;
  ...
</code></pre> 
            <p>Install fs dependency to your project:</p> 
            <blockquote> 
             <p>npm i fs --save-dev</p> 
            </blockquote> 
            <p>Run build:</p> 
            <blockquote> 
             <p>cordova build android</p> 
            </blockquote>

           <div class="post-text" itemprop="text"> 
            <p>It's a duplicate entry in values.xml in a folder called <code>support-compat-28.0.0-alpha1.aar</code>.</p> 
            <p>You'll find this file on Windows at <code>\users\YOURUSERID\.gradle\caches\transforms-1\files-1.1</code></p> 
            <p>Once inside that folder you have to go some levels deeper to <strong>values.xml</strong>.</p> 
            <p>In that file, search for an element <code>&lt;declare-styleable name="FontFamilyFont&gt;</code>.</p> 
            <p>In that element, remove the FOUR lines with <code>android:</code> in it.</p> 
            <p>After I made that change I can build again without errors.</p>
