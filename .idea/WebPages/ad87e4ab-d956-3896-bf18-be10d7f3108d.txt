          <div class="post-text" itemprop="text"> 
           <p>I'm coding a C program to read and parse a BMP header. The first 2 characters are chars, which I'm printing fine. However, all the other bits are ints in little endian format, so I'm storing them backwards. In hex, I'm getting the right answer. However, when I try to cast them to an int (for readability), I get an invalid number.</p> 
           <p>bmp.c --</p> 
           <pre><code>printf("file size: "%02x %02x %02x %02x\n", bmp-&gt;size[0], bmp-&gt;size[1], bmp-&gt;size[2], bmp-&gt;size[3]);

printf("file size: "%d\n", bit2int(bmp-&gt;size));
</code></pre> 
           <p>bit2int function --</p> 
           <pre><code>int bit2int(void *thing)
{
    return *(int*)thing;
}
</code></pre> 
           <p>output -- (actual file size is 415,798 bytes)</p> 
           <pre><code>file size: 00 06 58 36
file size: 911738368
</code></pre>

           <div class="post-text" itemprop="text"> 
            <p>BMP size is in <em>little endian</em>. So you're probably swapping the bytes, thus getting a big endian integer, which is of no use.</p> 
            <p>Then the cast you're doing is violating the strict aliasing rule <em>and</em> it's not portable.</p> 
            <p>Here's a small piece of code which tries to be as portable as possible. It reads the first 2 header bytes, then the 4 size bytes, and rebuilds the size, assuming the value is little endian:</p> 
            <pre><code>#include &lt;stdio.h&gt;
#include &lt;stdint.h&gt;

int main()
{
   FILE *f = fopen("foo.bmp","rb");
   if (f)
  {
   unsigned char header[3] = {'\0'};
   unsigned char size_buffer[4];
   int nb_read = fread(header,2,1,f);
   nb_read = fread(size_buffer,4,1,f);
   uint32_t size = size_buffer[0] + (size_buffer[1]&lt;&lt;8) + 
                   (size_buffer[2]&lt;&lt;16) + (size_buffer[3]&lt;&lt;24);
   printf("header: %s, size %lu\n",header,(unsigned long)size);
   fclose(f);
   }

}
</code></pre> 
            <ul> 
             <li>the code can be compiled on a big endian architecture without issues</li> 
             <li>the file is opened as binary, so it works on Windows &amp; Linux</li> 
             <li>I have omitted size checks when reading the header for simplicity. It can be added very easily.</li> 
            </ul>
